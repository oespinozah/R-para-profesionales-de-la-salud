# Contribuir al manual

Gracias por ayudar a mejorar este material. La prioridad del proyecto es que el manual sea claro, reproducible y útil para profesionales de la salud.

## Criterios de edición

- Escribir en español claro, directo y respetuoso.
- Mantener ejemplos con datos simulados o datos abiertos.
- Explicar la interpretación clínica, no solo el resultado técnico.
- Evitar rutas absolutas; usar rutas relativas al repositorio.
- No subir archivos de sesión como `.RData`, `.Rhistory` o cachés locales.
- Si agregas un paquete, inclúyelo en `scripts/instalar-paquetes.R`.

## Estilo de capítulos

Cada capítulo debe tener:

- Una pregunta o propósito práctico.
- Código reproducible.
- Interpretación de salidas.
- Advertencias sobre supuestos, sesgos o limitaciones.
- Referencias cuando corresponda.
